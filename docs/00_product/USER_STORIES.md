# User Stories

**Project:** In‑Play Predictor (CryptoScreener‑X) — ML + LLM  
**Date:** 2026-01-24

---


## Primary persona: опытный скальпер/интрадей трейдер

### Epic A — Быстро находить «торгуемые» сетапы
- **US-A1**: Как трейдер, я хочу видеть топ‑N символов, которые станут торгуемыми в ближайшие **30s / 2m / 5m**, чтобы не тратить внимание на «мёртвые» рынки.
- **US-A2**: Как трейдер, я хочу переключать **профиль исполнения** (limit-heavy / aggressive) и получать разный скор, чтобы подстраивать стиль.

### Epic B — Избегать ловушек и токсичного потока
- **US-B1**: Как трейдер, я хочу видеть `p_toxic` и статус **Hot but Dirty / Trap**, чтобы не входить туда, где высокий шанс adverse selection.
- **US-B2**: Как трейдер, я хочу видеть причины (reason codes) — «почему trap», чтобы доверять системе.

### Epic C — Прозрачность и контроль
- **US-C1**: Как трейдер, я хочу краткое объяснение (1–2 строки), чтобы быстро принять решение.
- **US-C2**: Как трейдер, я хочу видеть health‑статусы данных (stale book, missing streams), чтобы игнорировать неверные сигналы.
- **US-C3**: Как трейдер, я хочу настраивать пороги/веса и быстро проверять эффект через replay.

### Epic D — Оповещения
- **US-D1**: Как трейдер, я хочу получать Telegram-алерты только по событиям (вход в топ, TRADEABLE, TRAP), чтобы не получать спам.
- **US-D2**: Как трейдер, я хочу «анти‑фликер» (hysteresis), чтобы список не прыгал.

**Acceptance checklist:** для каждого US должен быть тест(ы), подтверждение контрактов и proof bundle.
